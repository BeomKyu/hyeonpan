## 지적 사항 (3건)

### 1. ⚠️ [중간] chat 탭 — v1 대화 기능 미통합

**spec.md L721-723:**
```
? active_tab == "chat":
  $Card.DetailModal.conversation_timeline(card)
```

**실제 구현 (DashboardDetailModal.tsx L90-96):**
```tsx
{activeTab === 'chat' && (
    <section className="dash-section">
        <div className="chat-placeholder">
            <p>💬 대화 기능은 v1 수동 모드에서 사용 가능합니다.</p>
            <p className="chat-hint">VITE_ADAPTER_MODE=manual 로 전환하세요.</p>
        </div>
    </section>
)}
```

**분석:** spec은 `$Card.DetailModal.conversation_timeline(card)` 재사용을 명시하지만, 실제 구현은 placeholder 메시지로 대체. Dashboard 모드에서 카드가 `$Model.Card`를 확장하므로 `messages` 배열이 빈 상태(`[]`)인 것은 맞지만, spec과의 차이가 존재한다.

**판정:** Spec 충실도 경미 위반. 단, Dashboard 모드 카드는 state.md 기반이므로 대화 기록이 없는 것이 논리적으로 자연스러움. **수정 불필요하나, spec 또는 코드 주석에 의도적 편차 기록 권장.**

### 2. ⚡ [경미] `findField` 파서 — 중첩 필드 오탐 가능성

**stateFileParser.ts L8-16:**
```ts
function findField(lines: string[], fieldName: string): string {
    for (const line of lines) {
        const trimmed = line.trim();
        if (trimmed.startsWith(fieldName)) {
            return trimmed.slice(fieldName.length).trim();
        }
    }
    return '';
}
```

**분석:** `findField("test_res:")` 호출 시, `meta:` 섹션 내부의 인덴트된 `test_res:` 줄도 `.trim()` 후 매칭됨 → **현재 동작이 올바름.** 그러나 만약 state.md에 최상위 `test_res_extended:` 같은 prefix 충돌 필드가 추가되면 오탐 발생 가능.

**판정:** 현재 state.md 포맷에서는 문제 없음. 미래 확장 시 정규식 기반 `fieldName + /:\s/` 패턴으로 교체 권장. **수정 불필요.**

### 3. ⚡ [경미] `useDashboard` — stale closure 가능성

**useDashboard.ts L76-78:**
```ts
const handleAddManual = useCallback((label: string, content: string) => {
    const updated = addManualProject(projects, label, content, '');
    refreshBoard(updated);
}, [projects, refreshBoard]);
```

**분석:** `projects` 가 deps에 포함되어 있어 매 프로젝트 추가마다 callback이 재생성됨. 빠른 연속 추가 시 stale closure 이론적 가능성 있으나, 실사용 패턴(UI 버튼 클릭)에서는 발생 확률 극히 낮음.

**판정:** React best practice 관점에서 `useReducer` 패턴이 더 안전하나, 현재 규모에서 과도. **수정 불필요.**
